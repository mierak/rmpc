An array of `Property` objects which serve as a template for table of songs thathat are currently in the queue. Each
property is a `song_table_format`.

## song_table_column

Describes a single column in the song table. This is a recursive data structure. Whenever a property is not defined the
`default` fallback is used. This fallback can also have a default value.

### prop

<ConfigValue name="prop" type="other" customText="<property>" link="../properties" />
Property of the song to display in the column. Described in <a href={path("configuration/properties")}>its own page</a>.
Only `Song` properties are allowed here and thus the property type is omitted here.

<ConfigValue
    name="kind"
    type={[
        "Property(File)",
        "Property(Filename)",
        "Property(FileExtension)",
        "Property(Title)",
        "Property(Artist)",
        "Property(Album)",
        "Property(Duration)",
        "Property(Disc)",
        "Property(Position)",
        'Property(Other("<tag_name>"))',
    ]}
/>

These will display value of the respective tag from the song. Since song files can have arbitrary tags,
the `Other` variant can be used to display any tag supported by MPD, even those not explicitly supported by rmpc.

### width

<ConfigValue name="width" type="string" />
Portion of the table width reserved for this column. Can be one of:

- Exact value: `"5"` which will result in 5 column width
- Percentage: `"50%"` which will result in half the table width

### alignment

<ConfigValue name="alignment" type={["Left", "Right", "Center"]} />
Percentage of the table reserved for this column. Sum of these value must be 100.

### label

<ConfigValue name="label" type="string" optional />
Optional label for the column. Displayed in the table header. Name of the property is used if not provided.

## Example

This configuration displays a table with a single column.

This column displays Artist of the song followed by a dash and Title of the song. If either the
artist or title is missing in song's metadata its Filename is shown instead.

```rust
song_table_format: [
    (
        prop: (
            kind: Group([
                (kind: Property(Artist)),
                (kind: Text(" - ")),
                (kind: Property(Title)),
            ]),
            default: (kind: Property(Filename))
        ),
        width: "100%",
    ),
]
```
